Trois missions réalisées en alternance, mises en correspondance avec les compétences du référentiel BTS SIO SISR.

SP 01

Sauvegarde et restauration des postes de formation

Microsoft Backup Entra ID Windows 10/11 3 jours
B2 B3
Contexte & Problématique

Les sessions de formation se succèdent régulièrement à INGETIS. Avant chaque nouvelle session, les postes doivent être remis dans un état propre et identique pour tous les apprenants. Cette opération, réalisée manuellement poste par poste, était longue et sujette aux erreurs de configuration.

Objectif

Automatiser et fiabiliser la remise en état des postes via un système de sauvegarde/restauration, pour garantir des environnements conformes et disponibles à chaque session sans intervention manuelle répétitive.

Réalisations
  • Configuration d'un poste "Master" avec les logiciels et paramètres types d'une session
  • Création d'une sauvegarde complète via Microsoft Backup (état système + données)
  • Déploiement de la sauvegarde sur l'ensemble des postes de la salle
  • Intégration des comptes de formation dans Microsoft Entra ID
  • Tests de vérification des accès et de la conformité des postes
Résultat

Préparation des salles en moins d'une heure au lieu d'une demi-journée. Zéro incident de configuration sur les sessions suivantes. Les formateurs peuvent enchaîner les sessions sans intervention technique.

Compétences référentiel :
B2.1 — Gérer le patrimoine informatique B2.3 — Assurer la continuité de service B3.1 — Installer et configurer un poste Administration Cloud Microsoft Support aux utilisateurs (formateurs)
SP 02

Enrôlement et déploiement MDM des postes — Belleville Citoyenne

Microsoft Intune Autopilot Entra ID 5 jours
B2 B3 B4
Contexte & Problématique

Dans le cadre du renouvellement du parc informatique de l'association, plusieurs postes devaient être préparés, sécurisés et remis aux collaborateurs rapidement. La configuration manuelle individuelle n'était pas envisageable à cette échelle, et chaque poste devait respecter la politique de sécurité de l'organisation.

Objectif

Industrialiser le déploiement via une solution MDM (Mobile Device Management) pour garantir une configuration uniforme, sécurisée et reproductible, sans intervention manuelle sur chaque machine.

Réalisations
  • Identification des machines et import des numéros de série dans Microsoft Intune
  • Création des profils de déploiement Autopilot (configuration OOBE zéro-touch)
  • Paramétrage des politiques de sécurité : BitLocker, restrictions applications, mise à jour
  • Attribution automatique des licences M365 via Entra ID
  • Tests de conformité sur le tableau de bord Intune (statut, applications, droits)
Résultat

Postes opérationnels dès la première connexion utilisateur, sans intervention technicien sur site. Réduction du temps de déploiement de 80%. Tous les postes sont conformes à la politique de sécurité de l'association.

Compétences référentiel :
B2.1 — Gérer le patrimoine informatique B3.2 — Déployer un système d'exploitation B4.1 — Protéger les données Modern Management (MDM/MAM) Politiques de sécurité Intune

Réalisation N°2 — Réseau

Segmentation réseau avec VLANs pour sécuriser l'infrastructure

Cisco IOS Router-on-a-Stick VLAN 10 / 20 DHCP Packet Tracer
B3 B4
Contexte & Problématique

Mediacom SARL dispose d'une infrastructure réseau à plat sans segmentation : les Techniciens et les Développeurs partagent le même réseau, ce qui pose des problèmes de sécurité et de cloisonnement des données. La mission consiste à segmenter le réseau en VLANs distincts tout en maintenant un routage inter-VLAN maîtrisé.

Architecture mise en place
ÉquipementInterfaceAdresse IPRôle
Router5Fa0/0.10192.168.10.1/28Passerelle VLAN 10
Router5Fa0/0.20192.168.10.17/29Passerelle VLAN 20
SW1Fa0/1 trunk—Lien trunk → Router5
SRV DHCP—192.168.10.21/29Distribution IP auto
PC-A (Tech.)VLAN 10DHCP .3→.14Postes Techniciens
PC-B (Dev.)VLAN 20DHCP .19→.22Postes Développeurs
Réalisations
  • Conception du plan d'adressage IP : VLAN 10 192.168.10.0/28 (Techniciens) et VLAN 20 192.168.10.16/29 (Développeurs)
  • Configuration du switch Cisco 2960 (SW1) : création des VLANs 10 et 20, attribution des ports en mode access, configuration du port trunk vers le routeur
  • Configuration du routeur Cisco 2621XM (Router5) : création des sous-interfaces Fa0/0.10 et Fa0/0.20, encapsulation dot1Q, routage inter-VLAN via la méthode router-on-a-stick
  • Configuration du serveur DHCP : 2 pools d'adresses (un par VLAN), attribution automatique des IPs aux postes clients
  • Tests de connectivité : ping inter-VLAN, vérification attribution DHCP sur chaque poste
Résultat

Infrastructure réseau segmentée et opérationnelle. Les deux VLANs communiquent uniquement via le routeur (contrôle du trafic). L'attribution DHCP automatique est fonctionnelle sur les deux sous-réseaux. Tests de ping inter-VLAN validés avec succès.

Compétences référentiel :
Concevoir une solution d'infrastructure réseau Installer, tester et déployer une solution réseau Exploiter, dépanner et superviser une solution réseau Cisco IOS — VLANs & Trunk Routage inter-VLAN (Router-on-a-Stick) DHCP multi-pool

Réalisation N°1 — Système

Modernisation de l'infrastructure — Migration Active Directory & Virtualisation des serveurs

VMware ESXi 8.0 Windows Server 2022 Active Directory DNS / DHCP Partages NTFS
B2 B3 B4
Contexte & Problématique

Mediacom SARL (28 utilisateurs) dispose d'une infrastructure informatique obsolète sans gestion centralisée des utilisateurs. La mission consiste à moderniser complètement l'infrastructure en déployant une solution virtualisée hébergée en datacenter, avec un annuaire Active Directory, un serveur de fichiers sécurisé et un système de sauvegarde automatisé.

Architecture déployée
ServeurAdresse IPRôle
ESXi (DC)10.0.0.6/24Hyperviseur hébergé
SRV-DC01192.168.10.10/24AD DS · DNS · DHCP
SRV-FILES01192.168.10.11/24Serveur de fichiers
SRV-BACKUP01192.168.10.12/24Sauvegarde auto.
Postes clientsDHCP .100→.20028 postes Windows 10/11
Réalisations
  • Déploiement de VMware ESXi 8.0 hébergé en datacenter (IP : 10.0.0.6) avec accès distant sécurisé via vSphere Client
  • Création de 3 VMs Windows Server 2022 : SRV-DC01, SRV-FILES01, SRV-BACKUP01
  • Configuration d'Active Directory (domaine entreprise.local) avec 28 comptes utilisateurs, DNS intégré, DHCP (plage .100→.200)
  • Déploiement d'un serveur de fichiers avec partages réseau SMB, permissions NTFS par groupe et quotas par utilisateur
  • Mise en place de la sauvegarde automatisée (Windows Server Backup) planifiée à 2h du matin, rétention 7 jours
  • Configuration de l'interconnexion sécurisée entre le réseau datacenter (10.0.0.0/24) et le réseau local (192.168.10.0/24) via routage statique et VPN
Résultat & Validation

Infrastructure complète opérationnelle : 28 comptes AD fonctionnels, authentification et résolution DNS validées, accès aux partages réseau vérifiés par groupe, sauvegardes testées avec restauration réussie. Latence inter-sites <50ms. Zéro interruption de service pendant la migration.

06 — Veille Technologique

Actualités & Tendances

Thème de veille : Microsoft Cloud, gestion des identités et cybersécurité des infrastructures modernes.

Janv. 2025Déploiement

Autopilot Device Preparation : le déploiement Windows encore simplifié

Microsoft a introduit une version modernisée d'Autopilot qui réduit significativement le temps de provisioning des postes Windows. Les erreurs de déploiement sont mieux gérées et la configuration est plus rapide pour les administrateurs.

Source : Microsoft Tech Community
Déc. 2024Sécurité

Entra ID : généralisation de l'authentification sans mot de passe (Passwordless)

Microsoft pousse les entreprises vers les méthodes sans mot de passe — FIDO2, Windows Hello for Business — pour renforcer la sécurité des identités. Ces méthodes résistent aux attaques de phishing contrairement aux mots de passe classiques.

Source : Microsoft Security Blog
Nov. 2024IA & Sécurité

Microsoft Copilot for Security : l'IA au service des administrateurs

Microsoft intègre l'IA générative dans ses outils de sécurité. Copilot for Security permet d'analyser rapidement les incidents, d'interpréter les logs et de proposer des remédiations, réduisant le temps de réponse aux menaces.

Source : Microsoft Ignite 2024
Oct. 2024Windows

Windows 11 24H2 : nouvelles politiques de conformité Intune

La mise à jour majeure de Windows 11 introduit de nouvelles capacités de conformité pour Intune : chiffrement BitLocker renforcé, protection contre les ransomwares améliorée et meilleures intégrations avec les règles de sécurité conditionnelle.

Source : Microsoft Docs
Sept. 2024Modèle

Zero Trust : le standard de sécurité qui s'impose dans les DSI

Le modèle Zero Trust — "ne jamais faire confiance, toujours vérifier" — est devenu la référence pour sécuriser les accès en environnement cloud. Microsoft 365 Defender et Entra ID s'alignent entièrement sur ce paradigme avec les accès conditionnels.

Source : ANSSI / CISA / Microsoft
Août 2024Exchange

Fin de l'authentification basique sur Exchange Online

Microsoft a définitivement désactivé les protocoles d'authentification classiques (POP, IMAP, SMTP Auth sans OAuth) sur Exchange Online. Les entreprises doivent migrer vers des méthodes modernes et plus sécurisées basées sur OAuth 2.0.

Source : Microsoft Exchange Team Blog
